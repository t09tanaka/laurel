# Editor integrations

Laurel ships a VS Code helper extension under `editors/vscode/` with config
schema support, snippets, and lightweight file associations. Other editors can
reuse the same generated JSON schema files, but Laurel does not currently ship
dedicated Helix, Neovim, or Obsidian plugins.

## Helix / Neovim

Use the generated schemas directly:

- Config: `editors/vscode/schemas/laurel.config.schema.json`
- Frontmatter: generated by `laurel schema frontmatter`
- Theme package metadata: generated by `laurel schema theme`

For Helix or Neovim, wire those schemas through your YAML / TOML language
server of choice. Keep the schema path relative to the repository root so it
works in devcontainers and Codespaces.

## Obsidian

Obsidian can edit `content/posts/*.md` and `content/pages/*.md` as normal
Markdown files. Laurel does not publish an Obsidian publish plugin today.
Recommended workflow:

1. Keep the vault inside the Laurel repository or sync selected notes into
   `content/posts/`.
2. Use Laurel frontmatter fields (`title`, `slug`, `date`, `tags`, `authors`,
   `status`) instead of Obsidian-only publish metadata.
3. Run `laurel check --check-frontmatter` before publishing.
