# Editor integrations

Nectar ships a VS Code helper extension under `editors/vscode/` with config
schema support, snippets, and lightweight file associations. Other editors can
reuse the same generated JSON schema files, but Nectar does not currently ship
dedicated Helix, Neovim, or Obsidian plugins.

## Helix / Neovim

Use the generated schemas directly:

- Config: `editors/vscode/schemas/nectar.config.schema.json`
- Frontmatter: generated by `nectar schema frontmatter`
- Theme package metadata: generated by `nectar schema theme`

For Helix or Neovim, wire those schemas through your YAML / TOML language
server of choice. Keep the schema path relative to the repository root so it
works in devcontainers and Codespaces.

## Obsidian

Obsidian can edit `content/posts/*.md` and `content/pages/*.md` as normal
Markdown files. Nectar does not publish an Obsidian publish plugin today.
Recommended workflow:

1. Keep the vault inside the Nectar repository or sync selected notes into
   `content/posts/`.
2. Use Nectar frontmatter fields (`title`, `slug`, `date`, `tags`, `authors`,
   `status`) instead of Obsidian-only publish metadata.
3. Run `nectar check --check-frontmatter` before publishing.
